1 · 背景:continuous vs discrete diffusion LM 的对立

1.1 简史 — 从 D3PM 到 MDLM 的"离散派"凯歌

扩散 LM 这条线 2021 年从 Austin et al. 的 D3PM (NeurIPS 2021) 起步,把图像扩散思想搬到 token,用离散 Markov chain。但真正引爆是 2023–2024 的 SEDD (Score-Entropy DM, Lou et al.) 和 2024 年的 MDLM (Sahoo et al., NeurIPS 2024) ——它们一起在 OpenWebText 上把离散扩散 LM 的 PPL 打到接近 AR baseline。Sahoo et al. 在后续工作 "Beyond Autoregression: Discrete Diffusion for Complex Reasoning and Planning" (引用 [51], 2025) 中给出了第一条严格 IsoFLOP scaling law:MDLM-low-var 对 AR 的 compute 鸿沟仅 14×,Duo (Schiff et al., 2025, 离散 + uniform noising 改写) 是 22–23×。这条 14× 数字成为整个 dLLM 社区"对标 AR"的事实标准。

1.2 "Continuous 不可扩展"是怎么变成共识的

另一条路 — 连续扩散 (continuous diffusion) — 把离散 token embed 到 ℝde,再在连续空间里 Gaussian noising,代表作:

  • Diffusion-LM (Li et al., NeurIPS 2022): 第一个把 Gaussian diffusion 接到文本,但只在 GeneralLM/ROCStories 等小规模 demo。
  • Plaid (Gulrajani & Hashimoto, NeurIPS 2023, 引用 [18]): 把 Plaid 作为 VDM (Kingma et al., NeurIPS 2021) 对文本的严肃实例,引入 可学习 embedding + 可学习 noise schedule + output prior + self-conditioning。但 Plaid 原报告与 AR 的 64× compute gap,几乎成了"连续派不行"的官方判决书。
  • CDCD (Dieleman et al., 2022, 引用 [15]) / FLM (Gao et al., 2024, 引用 [29]) / LangFlow (Chen et al., 2025, 引用 [10]) / DFM / CFM (Stark/Davis): 后续的连续派多采用 固定 one-hot embedding + 手工 schedule reparameterization + cross-entropy loss,但报告的 PPL 也始终落后离散派。

所以社区的工程直觉变成了:

"连续扩散对文本本质不合适,token 是离散的,在 ℝde 上做 score matching 信号噪声比天然差。"

本文的根本主张正面挑战这条直觉:差的不是范式,是历史代码栈没对齐。Plaid 原版用的还是 2022 年的 transformer 配方(RMSNorm,no AdaLN,no MLP bias,standard GELU),而 MDLM/Duo 都用了 DiT-style 现代 backbone(LayerNorm + AdaLN-Zero + RoPE + GELU(tanh) + MLP bias)。把这层架构差异填平,Plaid → RePlaid,scaling 鸿沟从 64× → 20×,瞬间和 MDLM 14× 同一量级。

三条偏见的工程根源(按论文的隐含归因):
① Architecture: 2023 Plaid 用旧 transformer,MDLM/Duo 用 DiT;
② Training recipe: Plaid 论文里 dataset / batch / 优化器和现代 dLLM scaling 文献完全不可比;
③ Model size: Plaid 原最大模型只跑到 ~100M,而 MDLM scaling 跑到 1.5B。
论文 §3.1 同时把这三层填平,再做 IsoFLOP。

1.3 三条路线的"哲学"差异 — 为什么仍然值得做连续

连续派被宣判但没死的原因(论文 intro 第一段反复强调):

能力离散扩散连续扩散

2 · Motivation:为什么要 revisit Plaid

论文的核心 thesis (intro 第 4 段) 可以浓缩成两句话:

(1) Plaid 原报告的 64× compute gap 并不是连续扩散的内禀性质,而是 "和现代离散 DLM scaling law 协议不可比" 的产物——dataset 不同 / 优化器不同 / 架构旧版。
(2) 在 Sahoo et al. [51] 完全一致的协议下重做 ——同 SlimPajama 数据集、同 Llama-2 tokenizer、同 AdamW + cosine、同 IsoFLOP 5 个 budget——把 Plaid 的 transformer 架构换成 MDLM/Duo 同款 DiT (其余算法不变),compute gap 立刻 "自动" 收窄到 20×。

这条 motivation 在叙事上极其纯粹:不发明新算法,不引入新 loss,只是 把变量 control 干净再测一次。所以论文一边给出新 SOTA,一边其实是在做一篇"benchmark 方法论"论文 — 它的二级 contribution 是告诉社区:未来比较连续 vs 离散扩散,必须在统一的 IsoFLOP scaling protocol 下做。

3 · Method 1 — Plaid 算法的最小回顾

RePlaid 保留 Plaid 的 全部 算法侧设计,只换 transformer 架构,所以理解 Plaid 是必要前提。这里只贴论文 §2 用得到的部分。

3.1 嵌入与前向过程

设词表 [V] = {1, ..., V}, 长度 L 的序列 x ∈ {0,1}L×V (每行 one-hot)。Plaid 维护一个 可学习 embedding 矩阵 E ∈ ℝV×de,每一行 单位 ℓ2 范数。嵌入序列 e = xE ∈ ℝL×de。论文 default de = 16,这是 RePlaid 与所有"高维 one-hot embedding"连续扩散方法 (FLM/CFM/DFM/LangFlow 默认 de=768) 在算力上的关键差。

前向过程是 VDM 的 variance-preserving Gaussian 扩散:

q(zt | x) = 𝒩(αt e, σt2 I) ,   αt2 + σt2 = 1 ,   SNR(t) = αt2/σt2 = e−γ(t)

γ(t) 是 monotone-increasing 的 noise schedule (γ(0) = γ0, γ(1) = γ1),参数化为

γ(t) = γ0 + (γ1 − γ0) · γ̃(t)

其中 γ̃: [0,1] → [0,1] 是一个 monotone MLP (Plaid 原作的做法),端点 γ0, γ1 始终可学习。

3.2 NELBO (训练目标)

反向过程 pθ(zs|zt) = q(zs|zt, x = xθ(zt, t)) — 把 denoiser 的预测插回 VDM 的闭式 Gaussian posterior。NELBO 三项加和:

−log pθ(x) ≤ ℒNELBO(x) = KL(q(z1|x) ∥ p(z1))
    + 𝔼z0∼q(z0|x) [ ∑l −log ⟨xθl(z0, 0), xl⟩ ]
    − ½ 𝔼t∼𝒰[0,1], zt∼q(zt|x) [ SNR′(t) · ∥eθ(zt, t) − e∥² ]

三项分别叫 prior loss / reconstruction loss / diffusion loss。其中 eθ(zt, t) := xθ(zt, t) E (denoiser 输出 categorical, 再投影回 embedding 空间)。

3.3 三个"Beyond VDM"的小件

Plaid 比纯 VDM 多三件 (RePlaid 全部保留):

  1. categorical reparameterization: eθ = xθ E (denoiser 不直接出 embedding,而是出 categorical,再左乘 E)。
  2. output prior logits: 在 softmax 之前,给 logits 加一个 closed-form Gaussian log-density,使预测倾向 zt 邻近的 plausible token。
  3. self-conditioning (s.c.): 训练时 25% batch 先做一遍 gradient-free forward,把 ẑ0 拼到输入再正式 forward;val/sample 时 always on。Tab. 2 显示 s.c. 贡献 ΔPPL = 1.5。

3.4 学习的两个统计量 (这是后面 §6 理论的伏笔)

γ0, γ1 通过最小化 diffusion loss (期望) 学到;γ̃(t) 通过最小化 diffusion loss 的 Monte-Carlo variance 学到。这套"分两路梯度"的训练方法直接从原 VDM (Kingma et al. 2021) 继承 — RePlaid 没改,但本文 §5.2 第一次给出了这件事 为什么 工作的闭式数学解释。

5.2 IsoFLOP & Scaling Law 拟合

对每个固定预算 C,扫不同参数量 N (Tab. 10 给出每个 C 下扫的 N grid),记录 validation loss L(N; C) (扩散模型用 VDM NELBO 上界)。对每个 C 在 log-log 空间拟合二次:

log L(N, C) ≈ aC (log N)² + bC log N + cC

取每条曲线最低点 (N*C, L*C) → 拟合 power-law scaling L*C ∝ C−α。这套 protocol 直接抄 Sahoo et al. [51] (Chinchilla-style)。论文验证 pipeline 的方法:先复现 MDLM (low var.) 14× 和 Duo ≈22× (vs Sahoo 报 23×),再算 RePlaid。

5.4 Over-trained crossover (论文最实用的发现)

compute-optimal 是 Chinchilla 视角的"理论最优 N"。但在生产里我们经常做 over-training(比如 Llama-2 7B 被训到远超 Chinchilla 比例),为的是把推理参数压小。论文 §3.4 报告:

  • 在 same model size 下,RePlaid (s.c.) 训到 ≈ 3.1× compute-optimal 时 loss 就能 match MDLM 训到 ≈ 6.9× compute-optimal 的水平 — 相当于 RePlaid 用一半多一点的 over-training 量就追上同尺寸 MDLM 的 over-trained 极限。
  • 这翻译到生产语言就是: "如果你想要一个小模型,RePlaid 比 MDLM 在 over-training 阶段省更多 compute"。

6 · Method 4 — 理论核心:Variance-minimization ⇒ Linear CE

这是论文第二个 contribution (intro 里 "Theoretical Insights (i)")。问 motivation:为什么 likelihood-based 训练(只有 NELBO,没有 hand-crafted CE-schedule reparameterization)能 work,而 LangFlow/FLM/DFM 一类必须手工 schedule?论文给出的答案分两步走。

6.1 Per-timestep diffusion loss 的统计

定义模型在 t 时刻的期望 diffusion loss:

ℓθ,γ(t) := −½ SNR′(t) · 𝔼qdata(x) q(zt|x) ∥eθ(zt,t) − e∥² = −½ SNR′(t) · MSEθ(SNR(t))

Kingma et al. (2021) 已经证明:diffusion loss 的总积分 𝔼t[ℓθ,γ(t)] 仅取决于 θ 和端点 γ0, γ1,与 γ̃(t) 的形状无关;而其 variance Vart[ℓθ,γ(t)] 仅取决于 γ̃(t)。本文 Proposition 1 在此基础上推进一步:

Proposition 1 (Constant per-timestep diffusion loss). 在端点约束 γ(0)=γ0, γ(1)=γ1 下,存在唯一一个 γ*(t),使得 ℓθ,γ*(t) ≡ κ ≥ 0 对所有 t 成立,进而 Vart[ℓθ,γ*(t)] = 0,其中 κ = 𝔼t[ℓθ,γ(t)]。

解读:variance-minimizing schedule 等价于"per-timestep 损失严格常数"。这恰好就是 LangFlow/FLM/DFM 一类工作显式追求的"denoising 难度均匀分布",但是 Plaid 派只通过 minimize Var 就 自动 拿到 — 不需要手工写一条 schedule reparameterization。

论文还给出 γ*(t) 和 κ 的 closed form (App. F 式 (31)/(32)),并实测发现 RePlaid 学出来的 γ̃(t) 几乎和 closed-form 重合 (Fig. 6b 的"learned vs theoretical")。把学的换成 closed-form 后,per-timestep loss 完美变常数 (Fig. 6a)。

6.2 Per-timestep CE 的分解 (Prop. 2)

但 diffusion loss 是 weighted MSE,并不直接对应"解码难度"。论文跟随 LangFlow,把 per-timestep CE 定义为

CEθ,γ(t) := 𝔼qdata(x) q(zt|x) [ ∑l −log ⟨xθl(zt,t), xl⟩ ]

在 Bayes-optimal denoiser θ* 假设下 (Def. 1) 用 I-MMSE identity (Guo–Shamai–Verdú 2005) 把 mutual information 和 MSE 联系起来,得到

Lemma 3 (Linear Information Decay). 在 θ*, γ* 下,I(e; zt) 关于 t 严格线性下降,斜率由 κ 决定。

这是非常漂亮的结论 — 它说"variance-min schedule + Bayes-optimal denoiser"等价于"信息以恒定速率从 e 流向 zt",一种最自然的 denoising 节拍。再代入 CE 即得:

Proposition 2 (Per-timestep CE Under Optimality). CEθ*,γ*(t) = (H(x) − I(e; z0) + κt) + C(x | zt)
其中 κt 是"linear-in-t trend",C(x|zt) := ∑l H(xl|zt) − H(x|zt) 是 conditional total correlation,单调递增于 t,取值 [0, C(x)]。

实测 (Fig. 6c):RePlaid 的 per-timestep CE 在训练中段就已经接近线性,即使残差 C(x|zt) 非零。这就是 abstract 里"naturally yields linear cross-entropy over time"的精确数学表达。

t = 0 ⇒ per-timestep CE (Prop. 2) t CE(t) linear-in-t trend (κt + const) + C(x|zt) 残差 (单调 ↑)

  1. 纯 NELBO 训出的 E 自发收敛到 低秩、POS-meaningful 的几何(用 spaCy 给每个 subword 打 POS tag,t-SNE 投影出现清晰簇)。
  2. 额外加一个 CE loss (∇ 只回传给 E 和 θ,γ 保持不变) 反而伤 PPL — 因为 CE 的梯度是"discriminative / separative",会把 E 撑开,破坏低秩结构。
  3. 这是论文反驳 CDCD/LangFlow/FLM 这类 "用 CE loss 替代 MSE 扩散 loss" 的理论依据:CE 训练 = 离散判别目标 ≠ likelihood 上界。NELBO 才是 true upper bound on −log pθ(x),它附带的 latent geometry 正则化是 CE 拿不到的。

7.1 与 LangFlow / FLM 的对比

论文 Fig. 13/14 进一步把 RePlaid (de=768) vs LangFlow (de=768 default) vs MDLM (de=768) 的 embedding spectrum 画在一起:

  • RePlaid 即使在高维也维持低秩 → PPL 更好;

这条对照说明:"低秩 embedding 几何对 likelihood 有好处"是连续派独有的现象,并不能简单照搬到离散派。

10 · Sampling — PFODE solvers

这是连续派相对离散派的独家优势:训好后可以用 image-diffusion 社区成熟的 PFODE 高阶 solver 加速采样。论文测了 4 个:

  • DDPM ancestral (stochastic) — 用 VDM 闭式后验 q(zs|zt, x = xθ(zt,t)),Plaid 原版的默认。
  • DDIM — 一阶 deterministic;
  • DPM-Solver++(2M) — 二阶 multistep,image diffusion 主力;
  • Heun — 二阶,但每步 2 NFE,实际换算 T 后会吃亏。

Probability-flow ODE:

żt = vθ(zt, t) = (α̇t − αt σ̇t/σt) · xθ(zt, t) E + (σ̇t/σt) zt

关键结论 (论文 Finding 3):

  • 低 NFE 区 (T ≤ 32): DPM-Solver++(2M) > DDPM > DDIM,二阶 solver 优势显著;
  • 高 NFE 区 (T ≥ 64): DDPM > 所有 deterministic solver,DDIM 收敛到 DPM++ 的水平 — 论文解释这是因为高阶 solver 在 high-NFE 时退化为一阶;
  • RePlaid (s.c.) DDPM @ T=1024 拿到 GenPPL ≈ 21,显著优于其他所有连续派和大部分离散派 (除了 high-NFE Duo)。

Self-conditioning 的副作用:在 T ≥ 512 时,RePlaid (s.c.) 的 unigram entropy 比所有 baseline 都低,因为每步把上一步的 ẑ0 当作 condition 喂回去,会把分布越锐化。论文用 GenPPL-entropy Pareto frontier (Fig. 3c) 说明:在 same GenPPL 下,RePlaid (no s.c.) 的 entropy 比 Duo、FLM 都好;但 RePlaid (s.c.) 的 entropy 反而最低。这是 trade-off,不是 bug。

12.1 论文自己承认的局限

  1. 20× 鸿沟仍在。即使从 64× 打到 20×,RePlaid 离 AR 仍有 ~一个数量级 compute 差距。生产部署时 AR 仍是 throughput-optimal 选择。
  2. 规模封顶。所有 scaling law 在 6e18 ~ 1e20 FLOPs 区间拟合,最大模型 ≈ 1.5B non-embedding。Conclusion 明确:"Investigating whether these scaling trends persist in the multi-billion-parameter regime where modern LLMs operate is a critical next step." — 没有 7B/13B 训练。
  3. only 2 datasets。OWT + LM1B,都是 web/news,不涉及 code/math/reasoning。zero-shot downstream 也没报。
  4. encoder 仍 token-level。论文 conclusion 提到可以换更强的 contextual encoder (sequence-level conditioning),但本文没做。

12.2 笔者补充的待验证点 (对联合作者的 critic)

  1. 20× 是否可以再降? 论文 ablation 显示 embedding learning 贡献 17.3 PPL,如果能用更强 embedding 初始化(如把 AR backbone 的 unembedding 矩阵作为 warm start),也许能进一步压。
  2. self-conditioning 在 high-T 把 entropy 压低(§10)— 这在长生成场景会不会变成质量天花板?需要更严格的 long-form generation diversity 评测。
  3. instruction tuning & RL 阶段 RePlaid 是否还有竞争力?论文没回答 — 这是 production 落地的关键缺口,毕竟现代 LLM 的 RLHF/RL-VR 都是基于 AR 的 sample + reward。连续扩散的 RL 训练机制(reward gradient through zt?)是一个 entirely open 的问题。
  4. vs LLaDA-8B / Dream-7B: 论文只跟"scaling law 论文 baseline" (MDLM/Duo) 比,没和这些公开的 8B 离散 dLLM 比;后者训练数据/规模都不同,但是社区实际感知的 SOTA。
  5. 22.1 PPL bound 的 NELBO tightness:K-IWAE 实验给出 K=32 时 22.11 vs NELBO 22.10,gap 极小,这条很有说服力。但 LangFlow 的 NELBO 论文 (Tab. 3 App. E) 报告了大幅波动,说明 ODE-based bound 对 design choice 敏感 — 比较时要注意。

14 · 附录:理论细节展开

14.1 Prop. 1 的闭式 γ*(t) (App. F 式 (31))

Variance-minimizing schedule 的存在性 + 唯一性证明依赖于 SNR(t) 是一个 strictly decreasing C¹ 函数,且 ℓθ,γ(t) = −½ SNR′(t) MSEθ(SNR(t))。Constraint 是 γ(0)=γ0, γ(1)=γ1,变分对象是 γ̃(t)。论文用变分 + 端点积分约束推出

γ*(t) = −log SNR−1( Φ−1( t · Φ(SNR0) + (1−t) · Φ(SNR1) ) )

其中 Φ(u) := ∫u∞ MSEθ(s)/s · ds (App. F 式 (31) 的等价形式)。Φ−1 是其反函数。κ 则由 端点积分(式 32)给出。直觉:γ* 在 MSE 曲线"陡峭"的 SNR 区间里安排更多 t-密度,以达到 per-timestep 损失常数化。

14.2 I-MMSE 与 Lemma 3

I-MMSE 恒等式 (Guo–Shamai–Verdú, IEEE Trans. Info. Theory 2005):

d/dλ I(X; √λ X + N) = ½ MMSE(X | √λ X + N),   N ~ 𝒩(0, I)

把 λ ↔ SNR(t) 代入,e 替换 X,zt 替换 noisy obs,可得 d I(e; zt) / d SNR(t) = ½ MMSE(e | zt)。当 Bayes-optimal denoiser 把 MMSE 压到等于 MSEθ*,且 Prop. 1 让 ℓθ,γ(t) ≡ κ,链式法则代入即得 d I(e; zt) / dt = − κ ⇒ I(e; zt) 严格线性下降于 t。这就是 Lemma 3 的核心机制,论文 App. G。

14.3 Prop. 2 的总相关性残差 C(x | zt)

总相关性 (total correlation) C(X) := ∑l H(Xl) − H(X),衡量"各分量联合分布相对独立分布的距离"。在文本里,它对应"token 之间的全局耦合",即语法/语义/篇章一致性。Prop. 2 说:

  • 当 t → 1 (近 pure noise): zt 几乎无信息,C(x | zt) → C(x) — 即条件总相关性接近 unconditional total correlation,这部分是 RePlaid 一定要"花心思学"的语言全局结构;
  • 当 t → 0 (近 clean): zt 已经透露大部分信息,C(x | zt) → 0 — token 之间在 zt 已知时 conditionally independent;
  • 所以 CE(t) 的非线性部分 — "tokens 之间的耦合度随 t 单调上升" — 是 语言结构本身的代价,不是 RePlaid 设计能消除的。

这是非常 elegant 的结果:RePlaid 用 NELBO 训练自动把"模型能优化的部分"(线性 trend) 安排得 evenly distributed across t,而"模型不能消除的部分"(语言耦合残差) 也单调可解释。这让作者可以 honest 地说"我们 evenly distribute denoising difficulty without case-specific reparameterization"。

14.4 为什么 LangFlow 的"CE-schedule 匹配"等价物可被替代

LangFlow [10] 显式做的事情:用 CE-rate 匹配 noise density (Chen, Zhao, Sun 等 2025) — 这是一种基于 information-gain rate 的 schedule reparameterization。论文 §5.2 的隐含吐槽:

  1. LangFlow 用 CE loss 训练,但 CE 不是 likelihood 上界 — 所以理论上不能保证 PPL 改进;实测 PPL 32.2,远差于 RePlaid 22.1;
  2. LangFlow 用手工 schedule 来"模仿"linear CE,但这是 case-specific(不同 embedding 维度需要不同 reparam);
  3. RePlaid 用 NELBO + variance-min schedule,自动 在任何 embedding 配置下都拿到 linear CE,而且 PPL 更好。

15 · 附录:Sampling Worked Example

下面是 RePlaid (s.c.) 在 T=64 步、L=1024 token 下采样的一段流程伪代码(为读者构建直觉,非论文逐字):

# 初始化: z_1 ~ N(0, I_{1024 x 16})  注意是 16 维, 不是 V=32001 维
z = sample_gaussian(L=1024, d_e=16)
z_hat0_prev = None  # for self-conditioning

# DDPM ancestral sampling
for k in range(T, 0, -1):
    t = k / T
    s = (k-1) / T
    # 前向 denoiser (带 self-cond 通路)
    x_hat = x_theta(z, t, sc=z_hat0_prev)   # categorical L x V
    e_hat = x_hat @ E                       # L x d_e, 反向 embedding
    z_hat0_prev = e_hat                     # 留给下一步 sc

    # VDM 闭式 posterior q(z_s | z_t, x = x_hat)
    alpha_ts = alpha[t] / alpha[s]
    sigma2_ts = sigma2[t] - alpha_ts**2 * sigma2[s]
    mu = (alpha_ts * sigma2[s] / sigma2[t]) * z + (alpha[s] * sigma2_ts / sigma2[t]) * e_hat
    var = sigma2_ts * sigma2[s] / sigma2[t]
    z = mu + sqrt(var) * randn_like(z)

# 末步 argmax 出 token (这步不计入 T)
final_logits = decoder_head(z)              # L x V
tokens = argmax(final_logits, dim=-1)       # L 个 token id

对比离散派 (MDLM): 每步是 categorical sampling on V=32001 dim,没有 PFODE 高阶 solver 选项;但每步可以利用 mask schedule 一次性 unmask 多个 token。RePlaid 每步是 16-dim Gaussian update,非常便宜,所以 T=1024 也跑得起 — 这正是图 3 GenPPL 在 high-T 持续改善的工程基础。

16 · 与本仓库 MTP / TOP 等 LM 方法线的横向对照

本仓库 models/ 系列除了 dLLM 还有大量 MTP / register / future-prediction 方向 (#14 MTP, #15 FastMTP, #16 L-MTP, #17 MuToR, #18 FSP, #10 TOP)。和 RePlaid 的关系:

  • 这条线全部是 AR 内的训练目标改造 (multi-token prediction, future summary prediction, token ordering rank loss),核心动机是"AR 的 next-token-only 目标 sample-efficiency 不够"。
  • RePlaid 是 跳出 AR,用扩散框架,根本不写 next-token 目标。两条路在"为什么 plain AR 不够"上有共识,但解法完全不同 — MTP 系列是"在 AR 上加未来 token loss",RePlaid 是"整条 trajectory 同步 denoising"。
  • 有意思的对照:RePlaid 的 over-trained crossover (§9) 是"训得越久,小模型越值"— 这和 MTP 系列追求 inference-time efficiency 异曲同工。但 RePlaid 通过 likelihood 训练 + scaling-law 性质拿到,MTP 通过 head 结构改造拿到。

关键 takeaway 速记

① 翻案叙事: Plaid 64× 是架构 + 协议的伪命题;同 DiT 同 SlimPajama 同 IsoFLOP 下,连续派 (RePlaid) 20× 几乎追平 MDLM 14×。
② 5 处架构改: RMSNorm→LayerNorm · no-bias→bias · std-GELU→GELU(tanh) · no-AdaLN→AdaLN-Zero · fixed-residual→learnable AdaLN gate · 外加 FP32 logit head → bfloat16。
③ 两条理论: (Prop. 1) variance-min γ̃ ⇒ per-step diffusion loss ≡ κ;(Prop. 2) Bayes-optimal denoiser 下 per-step CE = (H(x) − I(e;z0) + κt) + C(x|zt) — 即 linear-in-t 主项 + 单调残差。
④ 主驱动: Tab. 2 消融把 embedding learning 列为最大贡献者 (Δ=17.3 PPL),NELBO 训出的 E 自发低秩 (de=16 时前 6 PC 解释 90% var) + POS-meaningful。CE loss 反而会破坏这个几何 (PPL 22.1 → 26.1)。
⑤ 实测优势: OWT @1M PPL 22.1 (≪ MDLM-low-var 23.1, ≪ Duo 25.2, ≪ LangFlow 32.2, ≪ Plaid 原 24.4);参数比 MDLM/Duo 少 1.8×;over-train 时同 N 用 3.1× compute 即追上 MDLM 6.9× 的水平。
⑥ 仍有 20× compute gap vs AR — 翻案不是"超过 AR",只是把连续派搬回 likelihood scaling 的牌桌。
