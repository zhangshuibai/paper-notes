1 · Motivation 与论文三问

1.1 论文 abstract (verbatim)

We introduce Nemotron-Labs-Diffusion, a tri-mode language model (LM) that unifies AR, diffusion, and self-speculation decoding within a single architecture. Trained with a joint AR-diffusion objective, Nemotron-Labs-Diffusion can switch modes to sustain high throughput across deployment settings and concurrency levels. … (1) AR and diffusion objectives are complementary: diffusion improves lookahead planning, while AR provides left-to-right linguistic priors. (2) In self-speculation mode, diffusion drafts while AR verifies, outperforming MTP methods in both acceptance rate and real-device efficiency. (3) A speed-of-light analysis further demonstrates diffusion's long-term potential, with up to 76.5% more tokens per forward pass than self-speculation under an optimal sampler.

2 · 背景: diffusion LM 当前 scaling 困局

2.1 离散派的 8B 平台 — LLaDA / Dream / SDAR 与 30-55% 区间

到 2025 年底,公开 8B-class 离散扩散 LM 主要是 LLaDA-8B (Nie et al.), Dream-7B (Ye et al.), SDAR-8B Chat (Cheng et al.),它们在 instruct benchmark 平均精度落在 37–54% 区间(见后面 Tab 5),而同尺度的 Qwen3-8B AR 已经做到 62.75%。这条 "差 ~10 pp" 的 gap 是过去一年 diffusion LM 社区最焦虑的事 — 因为 6× TPF 的并行优势,在精度落后 10 pp 的现实下没人敢用作主线部署。

2.2 连续派难 scale 的根因 (链接 #23 RePlaid)

连续 Gaussian diffusion 在 ℝd_e 上做 score matching,信噪比天然差,且早期 Plaid 报告对 AR 是 64× compute 鸿沟。RePlaid (本人参与) 把这个 gap 推到 20×,但本论文 (Nemotron-Diff) 选了完全相反的路 — 直接在离散 mask diffusion 上做工程化,而不在 score matching 上博 likelihood。两条路在 2026 H1 形成对照实验。

2.3 MTP / Eagle3 / Block Diffusion 三条 acceleration 的当前限制

路线代表核心限制
MTP (auxiliary heads)Gloeckle (#14) / Eagle3 / FastMTP (#15)draft 头容量小,远端 horizon 退化;Eagle3 仍要付 embedding+LM head 成本

Nemotron-Diff 的策略是 — 在 (joint AR-diff training) + (self-speculation 用 same-model AR 当 verifier) 两层都做改良,把三条路线的优点都吃进来。

3 · Method 1 — Joint AR-Diffusion 训练

3.1 三模式总览 (图 1)

3.2 双 Loss 数学定义

对一个 token 序列 x,论文同时优化两个 loss:

ℒ_AR(θ) = E_{x∼𝒟} [ −Σᵢ log p_θ(xᵢ | x_{<i}) ] …Eq.(1)
ℒ_diff(θ) = E_{t∼𝒰[0,1], x̃ᵇ_t∼q(·|xᵇ)} [ −(1/t) · Σᵇ log p_θ(xᵇ | x̃ᵇ_t, x_{<b}) ] …Eq.(2)
ℒ(θ) = ℒ_AR(θ) + α · ℒ_diff(θ) with α = 0.3 …Eq.(3)

其中 diff loss 是 block-wise(把序列分成 B 个块,只对当前块加 mask 噪声,前缀保持 clean),并带 1/t 的 reweighting — t 越小,被 mask 的 token 越少但每个权重越高(后面会看到这正是 global vs sequence avg 之争的根)。

3.3 双流 Attention 模式 — 图 2 详解

严格 causal mask (本文与 [9,10,13] 关键区别) ↓ 同一 forward 算 AR loss 无标签泄漏 ④ Clean→Noisy 禁用 (N/A) clean 不能看到 noisy 否则 AR loss 泄漏 mask 信息

3.7 Global vs Sequence Loss Averaging 数学对比

§2.1 单独花一节讲 loss aggregation,对应 Eq.(4)/(5):

ℒ_seq-avg = (1/N) Σ_n (1/L) Σ_i ℓ_{n,i} …Eq.(4)
ℒ_global = (1/(NL)) Σ_n Σ_i ℓ_{n,i} …Eq.(5)

当每条 sequence 有相同 mask 数时两者等价。但 diffusion loss 因为 1/t reweighting + mask token 数 ∝ t,会导致 每条 sequence 的有效 token 数差异大。Sequence-wise 平均会让"少量高权重"的小 t 样本被等同对待,显著增大 batch 间方差。Global avg 等价于按 contribute token 数加权,相当于天然的 importance reweighting。论文的对照实验显示这一项单独贡献 +2.12% — 是除 AR loss 和 two-stage 之外最大的一项。这条经验类比于 GRPO vs DAPO 的 RL aggregation 之争。

4 · Method 2 — Tri-Mode Inference

4.1 Mode 1: AR Decoding

最简单 — 模型作为标准 causal LM,x_i ~ p_θ(·|x_{<i})。这是因为 stage 2 训练时双流的 clean 流就是严格 causal,所以模型本身就有完整 AR 能力。什么时候用:high concurrency cloud serving,因为大 batch 下 memory bound 自然消解,parallel decoding 收益不大。

4.2 Mode 2: Block-wise Diffusion Denoising

Confidence-based sampling

沿用 Fast-dLLM [16] + Efficient-dLM [10] 的范式 — 一个块一个块解,块内所有位置初始化为 [MASK],按 confidence threshold 决定每步并行接受哪些位置。一个块解完后刷新 KV cache,进入下一块。

Trained sampler (Appendix A)

论文额外训了一个 4-layer 轻量 transformer (d=384, 4.8M params, ~0.06% backbone),输入是 144 维 per-position 特征(top-3 PCA 语义 embedding + top-1 prob + margin + entropy + top-3 mass),sigmoid 输出"top-1 是否会与最终 commit 一致"。训练数据从 8B 模型自身收集 ~20M denoising trajectories(B∈{8,32},math/code/STEM/chat)。这是 Pareto frontier 提升 — 同精度 1.3× TPF 或同 TPF +10.6% acc。

4.3 Mode 3: Linear Self-Speculation — 图 5 详解

✗ first reject x̂₅ discarded 本步 commit: 3 个 accepted prefix + 1 个 free verified token (x_AR₄) = 4 tokens 下一轮以新 verified prefix 为基础继续 → 每轮产出 1 到 k+1 个 token / 2 个 forward(draft + verify)

4.4 LoRA + LK-hybrid loss 强化 drafter

这里的工程巧思有几层:

  • LoRA 位置: 只加在 attention 的 o_proj 上(rank=128, α=512, ~36M)。这极小的 ~0.4% 参数把 8B 模型的 acceptance length 拉 +1.36(5.46→6.82, +25%)。
  • Active mask "accepted + 1": 只在 accepted prefix + 第一个 reject 位置算 loss(j* 之后被 mask 掉)。理由是 verifier 的 KV cache 在 reject point 之后会重建,逻辑上不再有意义 — 在那些位置训练会引入 counterfactual bias。
  • LK-hybrid loss (Eq. 6, 来自 Samarin et al. [19], 即 #spec-rl 14 的 LK Losses 论文):用 union top-K=200 截断 KL + TV(total variation) 双权重 — 早期 (α_j 接近 0) 像 forward KL,晚期 (α_j → 1) 像 TV(直接对 acceptance rate gap 求梯度)。
  • CE 项 (Eq. 7): 加一个 token-level CE against verifier argmax,作为强 teacher forcing。τ=1.0 不缩放。
  • Total: ℒ = ℒ_LK + ℒ_CE。Inner-batch 90% 用 softmax sampling,10% greedy — 让 drafter 见到更广的 verifier 输出分布。

4.5 Quadratic Self-Speculation (§3.4 变体)

把 draft 和 verify 合并到 单个 forward,用一个特殊 attention mask: causal 部分验证上一步的 draft,parallel diffusion 部分为下一步产生新 draft。代价是 attention 矩阵 O(k²) — 这是为什么叫 quadratic。优势是 always 产出 k 个 token / 1 forward(即使 reject 早发生)。论文进一步引入 AR-diffusion ensemble verifier(causal + diff 加权)。但 Tab 5 显示 Quad SS 实际部署落后 Linear SS — 因为 FlexAttention kernel 对这个 dedicated mask 优化不充分(详见后面 §12 局限)。

⭐ Quadratic SS 的方法学起源 = TiDAR (arXiv:2511.08923)。论文 §3.4 verbatim: "following the same process as [20]",其中 [20] 就是 TiDAR — 同 NVIDIA 团队 6 个月前的前作。Yonggan Fu 在 TiDAR 是 co-author,在 Nemotron-Diff 升任一作;Jan Kautz + Pavlo Molchanov 两篇都是 senior author。完整对比见 §11.5 与 TiDAR 的关系。

5 · Speed-of-Light 分析 (§4)

5.1 SOL 是什么 — 论文最重要的方法论贡献

Speed-of-Light(SOL) 是论文最有 ambition 的一节 — 它问的不是"我现在跑多快",而是"如果有最优采样器,diffusion 模式最快能多快?"。SOL 不涉及 AR verifier,完全在 diffusion 自己内部 — 它给当前所有 sampler / verifier 设计提供了一个 硬性 ceiling。

5.2 Oracle target — Serial denoising 定义"converged output"

对块长 B,从全 mask 输入 [M]^B 出发,每步找当前所有 mask 位置中 peak probability 最高的一个,commit 它的 argmax,重新 forward,重复 B 次。这是 B 个 forward / 一个 token / 完全保守的"如果不并行决策时模型最终会输出什么"。结果记为 target t = (t₁, …, t_B)。SOL acceptance ratio 就定义为"在平均多少个 forward 内能用某种并行策略从全 mask 重建出 t"。

5.3 Greedy parallel acceptance vs Recursive dynamic compaction

Greedy: 每个 iteration 看哪些位置的预测 == t_j,全部 commit。简单,但 "context dependent token 同时 commit" 会扰动剩余位置的条件分布,所以不保证收敛到 t — 这是个 lower bound,不是真正 SOL。

Recursive dynamic compaction(图 6 详解): 把 N 个 matched 位置按 confidence 排成 (p₁, …, p_N),做二分搜索找最大的 safe prefix —— "safe" 的定义是,commit 这个 prefix 后剩余位置用 greedy 解仍能到达 t(用 5000-forward budget 模拟验证)。每个 safety check 会递归调用 decoder 一次。Top-1 一定 safe(等价于 serial step),所以至少每步 commit 1 个 — 严格 dominate greedy。

5.4 图 6 — Recursive Dynamic Compaction 流程

5.6 SOL 7.60× vs Linear SS 6.82× vs 实际 3.41× — 76.5% headroom 含义

三组数字的逐步差异:

  • SOL acceptance rate 7.60× — 理论上限。1 forward 内可并行接受多少位置。
  • Linear SS acceptance rate 6.82×(LoRA 后)— 距 SOL 仅 10.3%。"AR 验证 diffusion draft" 是接近 SOL 的有效方式。
  • SOL real TPF 6.02× vs Linear SS real TPF 3.41× — 这是 76.5% gap!原因有二: (a) Linear SS 每轮 2 个 forward (draft + verify), real TPF = acceptance ÷ 2; (b) Linear SS 只接受 contiguous prefix,首个 reject 后所有"散落的对"全部丢弃。

76.5% 是这篇论文最有冲击力的一个数字 — 它告诉社区:不要满足于 Linear SS 的 6.82× acceptance,真正的并行性是 SOL 的 6.02 real TPF。要够到这个 ceiling,需要(论文 §8 "Future Directions" 提的)更智能的采样器、non-prefix 接受、diffusion-verify-diffusion 流水线。

  • multimodal projector: 2-layer MLP + 2×2 patch merging。
  • LM backbone: 直接用 Nemotron-Labs-Diffusion-8B Instruct 初始化 — 因为两套 ckpt 的 LM 架构相同,合并是 exact(no parameter mismatch)。
  • asymmetric dual stream(Eq. 10): noisy 流剥掉所有 vision token —— 因为 vision token 永远不会被 mask,放在 noisy 流里只是浪费 FLOPs。Clean 流保留完整序列。总序列长度 L_text + L 而非 2L。attention FLOPs 节约 ∝ N_vis/L,对高分辨率图像非常显著。
  • Continued SFT: full model (backbone + vision + projector) 用 joint AR-diff loss 在 FineVision [25] 上 fine-tune。

9.2 Asymmetric Dual Stream 工程巧思

Eq.(10):

[ x̃ᵗᵉˣᵗ,ᴸᵗᵉˣᵗ_t | x^L ], L_text = L − N_vis

普通双流 (text only) 把 sequence 复制 2 倍。VLM 如果直接复制,vision token 一半在 noisy 流(永不被 mask 仍 forward)— 纯浪费。Asymmetric 方案 strip vision token from noisy half,只保留 clean 流的完整序列(包括 vision token)用于 cross-stream conditioning。total length = L_text + L 而非 2L。attention FLOPs 节约 ∝ N_vis/L,高分辨率图(N_vis 占大头)收益最显著。

10 · 6 个 Takeaway + 4 个未来方向

10.1 六个 takeaway(§8 翻译)

  1. Tri-mode 是 joint training 的自然产物 — 不需要任何 mode-specific 架构修改。
  2. AR 和 diff loss 互补,不竞争 — 都在 α=0.3 同时 peak,AR loss 引入 left-to-right prior,diff loss 略提升 AR future planning。
  3. Self-spec 优于 MTP — 用 diffusion 自身做 drafter 而非 auxiliary head,acceptance 6.82 vs Eagle3 2.75。
  4. Variance reduction 对 diffusion 训练关键 — global loss averaging + two-stage AR init 都是降方差技巧。
  5. Linear SS 当前 infrastructure 最优 — Quad SS 单步 TPF 更高但 kernel 未优化,batch=1 部署仍要选 Linear。
  6. Diffusion 解码 headroom 巨大 — SOL 比 Linear SS 高 76.5% real TPF,sampler 还有大量优化空间。

10.2 四个未来方向

  1. 缩小 diffusion sampler 到 SOL 的 gap — 当前 confidence-based sampler 远未达上限,需要更智能的 sampler 或更激进的并行采样训练 scheme。
  2. 提升 draft-verifier 对齐 — LoRA 已经从 5.46 → 6.82,但用 nested subnets / Flextron / Amoeba-LLM 的 weight-share 子网做 drafter 还能进一步降本。
  3. 超越 prefix-only 验证 — 当前 AR verify 只接收 contiguous prefix,丢失了 diffusion draft 的"散落对"。未来或用 diffusion verifier 双方互检,接收非连续 token。
  4. 段级 / 段落级并行 — 当前 diffusion 解码仍偏 left-to-right,只是 token-level 并行。如果训练能鼓励 segment-level / paragraph-level 并行,会触及 diffusion 的全局规划真正本质。

12 · 局限 / 待考察

12.1 Quad SS 当前因 kernel 不优而落后 Linear SS

Tab 5 显示 Quad SS 单 forward acceptance 更高(6.38× TPF vs 5.99×),但 §6.1 明确说 "due to the use of FlexAttention with less optimized kernels for the dedicated attention mask, the real-device efficiency of quadratic self-speculation falls behind the linear one"。所以 Quad SS 是一个"未来 kernel 优化后会反超"的技术,目前部署默认仍是 Linear。这是工程现实 vs 算法潜力的典型张力。

12.2 Block size 越大,acceptance 越高但 accuracy 下降

Tab 4 末行 — BL=32 时 acc 61.81%,BL=4 时 acc 64.04%。差 2.23 pp。这是因为大 block 内多 token 同时 commit 时,context-dependent 关系被 "smear" 在并行采样里。如何在大 block 上保持精度,论文明确点为 "critical direction" — 这是后面"closing SOL-gap"研究的入口。

12.3 1.3T tokens continued pretraining 不便宜

Stage 1 1T + Stage 2 300B = 1.3T tokens 在 256 × H100 上跑两次。这超出了大部分学术 lab 的预算,academic 重现几乎不可能。对比 RePlaid (#23) 只用 <1B token IsoFLOP 就 prove 出 scaling law — 学术圈在 method 验证层面只能这样做。这也意味着 Nemotron-Diff 的具体 recipe(α=0.3, two-stage, 1T:300B 比例)是否唯一最优,缺少同等成本的对比实验。

12.4 没在 ≥30B 上验证 scaling

表 7 最大是 14B,而 LLaDA 2.0 已经做到 100B(论文引 [49])。Nemotron-Diff 的 "TPF 随 scale 上升"(3B 4.36→ 8B 5.99)这条 trend 是否能延伸到 30B+ 没有 evidence。一个 70B/100B 验证会让叙事完整,但目前仍是开放问题。

12.5 SOL 的 5000-budget 假设

Recursive dynamic compaction 的 "safe" 判断用 5000-forward simulation budget。如果 budget 改小,SOL 数字会下移;改大,SOL 数字会上移。论文没给 budget 敏感性分析 — 这意味着 7.60× 这个 SOL 数字本身依赖一个 implicit 的超参选择。

12.6 Trained sampler 的训练数据覆盖

Appendix A 说采样器训了 ~20M trajectories,来自 math/code/STEM/chat。但 SOL 11 类里 roleplay (3.49) / writing (6.13) / summarization (6.02) 都不在这 4 类里。"训过 4 类、评估 11 类" 的 sampler 是否对低相似类别可泛化,论文没明确实验。
